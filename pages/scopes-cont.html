More on scopes

Let's play with scopes a bit more. For this next example, lets pretend we're a bank and we're loaning some money out to a previous student who wants to open up their own AngularJS school (since clearly they are experts, now that they have completed my tutorial). We'll begin by defining some custom logic to determine the interest for our loan. Let's get moving!

Do you remember what's next? That's right! (or maybe it isn't... I can't actually hear you.) Let's head over to our main.js file and create a root controller for our banking bot. You can see mine below: 

app.controller('interestController', function($scope) {
  $scope.amt = 50000;
  $scope.terms = 1;
  $scope.rate= 10;
  $scope.interestamt= function() {
    return ($scope.amt*$scope.rate*$scope.terms)/100;
  };
});

Okay, so once again we need to make this available to the view. We hardcoded all of our banking data, so for now, lets just show the interest amount in a template: 

<div ng-controller="interestController">Interest Amount: {{ "$" + interestamt() }}</div>

Here it is: Interest Amount: {{ "$" + interestamt() }}

Cool! So that was wayyy to easy, though. Lets make this a little more user friendly. What if our customers want to know the interest for a loan with different parameters than we hard coded? The cool part is that we don't need to do any more work to our controller. We can make changes to the model right in the DOM! Let's create a table our users can interact with. Check out the code and finished product below. Play with the inputs to see the view update automatically.

<div ng-controller="interestController">
<tr><td>PRINCIPAL AMOUNT: </td><td><input type="number" ng-model="amt" ></td></tr>
<tr><td>RATE            : </td><td><input type="number" ng-model="rate"></td></tr>
<tr><td>TERMS		: </td><td><input type="number" ng-model="terms"></td></tr>
<tr>
<td></td><td>Interest Amount: {{ "$" + interestamt() }}</td>
<tr>
</table>
</div>

Pretty cool, huh? The view will immeadiately show our updated interest total. This is because of that weird "data-binding" thing we mentioned earlier. Don't worry if you're still a bit fuzzy on it, because that's up next!
