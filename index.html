Hello, {{ yourName }}. Welcome!

Welcome to In Depth Angular by Spencer Stone. In this tuitorial we will examine the inner workings of a very popular framework developed by Google, AngularJS. Enter your name in the above input to see angular at work.

So what is happening above, then? Bi-directional data binding. This essentially means changing data on the back end changes your view automatically. This is a very simple example of Angular at work. Let's take a deeper look into whats going on. Ready..Set..Code!

Lets see the code! Here are the elements we used in our app to make this happen:

  • ng-app
  • ng-model=“yourName”
  • {{ yourName }}

Wait...What is this stuff?

First, the ng-app attribute is added on the <html> tag. This tag allows Angular to start. Without this, the CDN used here is useless. It is usually added to root elements and could easily be applied to a body tag as well.

Second, we to set up bi-directional data binding on the yourName model simply by ading ng-model="yourName" to the input.

Lastly, we displayed the data yourName in the directive template called {{ yourName }}. If you are fimailiar with Handlebars or Mustache, you may be recgognize this template pattern.

This is about as simple as AngularJS gets. We have yet to even touch Javascript yet; but dont worry, we'll get to that as more customized behavior becomes necessary.

Javascript! Javascript!

Okay. You asked for it. Lets get started with the javascript. First we created a new JS file and linked it in our head tag. Next we need to define an angular.module. An Angular module is a collection of functions that are run when the application is started.

In our linked Javascript file, we will define our module as such: 

var app = angular.module('myApp', [])

The first arguement is the name of our module. The second, empty array will be adressed, shortly.

Next we must instantiate our myApp module and let Angular know where in the DOM tree to look for our app. Remember that ng-app directive we added earlier to the <html> tag? This is where we'll do this. It can be done by simply adding the name of our new module, like so: <html ng-app="myApp">.

Wherever ng-app is located in our HTML is where Angular will start. For our case, we want this to be on either the <html> or <body> tags, since we want the app to take over the entire page.
